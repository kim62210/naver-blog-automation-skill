```
[STEP 4/8] 결과 확인 ████████████████░░░░░░░░░░░░ 50%
```

---

## 4-1. 요약 포맷

```
📊 자료 수집 완료

[텍스트 자료]
• 네이버 뉴스: N건
• 네이버 블로그: N건
• 웹 검색: N건
• 총 수집: N건

[참고 이미지 - 다운로드 완료]
• 뉴스 이미지: N건
• 블로그 이미지: N건
• 이미지 검색: N건
• 총 다운로드: N건
• 저장 위치: ./images/

[핵심 정보 요약]
1. {핵심 정보 1}
2. {핵심 정보 2}
3. {핵심 정보 3}
...

[다운로드된 이미지 목록]
1. 📷 01_뉴스_{설명}.jpg - {유형}
   └ 출처: {출처 URL}
2. 📷 02_블로그_{설명}.png - {유형}
   └ 출처: {출처 URL}
...

[자료 충분성 판단]
✅ 충분 / ⚠️ 보통 / ❌ 부족

이대로 글 작성을 진행할까요?
1️⃣ 진행
2️⃣ 추가 검색 (키워드 지정)
3️⃣ 다른 주제로 변경
```

---

## 4-2. 자료 충분성 판단 기준

### ✅ 충분
- 텍스트 자료 15건 이상
- 핵심 정보 요약 5개 이상 도출
- 다양한 소스 (뉴스/블로그/공식사이트)
- 참고 이미지 8건 이상

### ⚠️ 보통
- 텍스트 자료 10~15건
- 핵심 정보 요약 3~5개 도출
- 일부 소스 편중
- 참고 이미지 5~8건

### ❌ 부족
- 텍스트 자료 10건 미만
- 핵심 정보 요약 3개 미만
- 소스 단일화
- 참고 이미지 5건 미만

## 4-3. 자료 부족 시

자료가 부족하다고 판단되면 사용자에게 알립니다:

```
⚠️ 수집된 자료가 충분하지 않습니다.

[부족한 부분]
- {구체적 내용}

어떻게 진행할까요?
1️⃣ 현재 자료로 진행 (내용이 얇을 수 있음)
2️⃣ 추가 키워드로 검색: "{제안 키워드}"
3️⃣ 다른 주제로 변경
```

---

## 4-4. 핵심 정보 추출

수집된 자료에서 글 작성에 필요한 핵심 정보를 추출합니다:

### 추출 항목
1. **핵심 개념**: 주제의 정의/개요
2. **중요 수치/데이터**: 통계, 금액, 비율 등
3. **비교 정보**: 장단점, 전후 비교
4. **실용 정보**: 신청 방법, 주의사항
5. **최신 동향**: 변경 사항, 트렌드

### 정보 구조화

```yaml
topic: "{주제}"
key_concepts:
  - "{핵심 개념 1}"
  - "{핵심 개념 2}"
data_points:
  - "{수치/데이터 1}"
  - "{수치/데이터 2}"
comparisons:
  - "{비교 정보}"
practical_tips:
  - "{실용 팁 1}"
  - "{실용 팁 2}"
recent_changes:
  - "{최신 동향}"
```
